## 1. 개발 목표

2026-07-24까지 스터디 영역을 완성한다.

2026-07-14까지는 핵심 기능을 1차 구현하고, 2026-07-15부터 2026-07-24까지는 예외 처리, UX 개선, 반응형, 테스트, 최종 QA를 진행한다.

| 목표 | 설명 |
| --- | --- |
| 스터디 목록 조회 | 페이지네이션, 검색, 포인트 순 정렬, 최신 순 정렬, 오래된 순 정렬을 제공한다. |
| 최근 조회한 스터디 | LocalStorage를 활용하여 최근 조회한 스터디를 표시한다. |
| 스터디 생성 | 닉네임, 스터디 이름, 소개, 배경, 비밀번호, 비밀번호 확인을 입력하여 스터디를 생성한다. |
| 스터디 상세 조회 | 스터디 정보, 응원 이모지 상위 3개, 포인트, 습관 기록표 영역을 조회한다. |
| 스터디 수정 | 비밀번호 검증 후 스터디 정보를 수정한다. |
| 스터디 삭제 | 비밀번호 검증 후 스터디를 삭제한다. |

## 5. 완료 기준

| 구분 | 완료 기준 |
| --- | --- |
| 생성 | 필수값을 입력하면 스터디가 생성되고 목록/상세에서 확인 가능 |
| 목록 | 페이지네이션, 검색, 포인트 순, 최신 순, 오래된 순 정렬 가능 |

| 예외 처리 | 빈 목록, 검색 결과 없음, 잘못된 비밀번호, 삭제된 데이터, 네트워크 실패 상태 처리 |
| QA | 생성, 목록, 검색, 정렬, 페이지네이션, 상세, 수정, 삭제, 최근 조회 주요 흐름 통과 |
